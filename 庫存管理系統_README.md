# 庫存管理系統

純前端庫存管理工具，主入口為 `index.html`。資料儲存在瀏覽器 `localStorage`，支援 Excel 匯入匯出、JSON 備份還原、出貨排程、BOM 扣庫、庫位管理與月報統計。

適合小型包材庫存、產品出貨扣庫、盤點與採購提醒。

## 專案結構

```text
index.html
css/style.css
js/data.js
js/analytics.js
js/inventory.js
js/location.js
js/shipment.js
js/bom.js
js/report.js
js/schedule.js
js/ui.js
檔案分工.md
```

各檔案詳細職責請看 `檔案分工.md`。

## 主要功能

- 總覽儀表板：庫存狀態、採購提醒、近期操作
- 庫存總覽：品項搜尋、篩選、新增、編輯、快速調整
- 出貨登記：產品扣庫，依 BOM 自動扣除包材
- 進貨登記：補充庫存並留下紀錄
- 批次盤點：比對帳面與實際數量
- 庫位管理：記錄品項所在庫位與庫位數量
- BOM 包材對照：設定產品出貨需要消耗的包材
- 出貨排程：月曆 / 列表視圖、今日提醒、確認出貨扣庫
- 包材使用率排行：依期間統計消耗量
- 庫存趨勢圖：查看品項出入庫趨勢
- 月報表：統計每月出貨、進貨與成本
- 採購清單：依庫存與週期產生採購建議
- Excel 匯入匯出與 JSON 備份還原

## 使用方式

建議用 VS Code + Live Server 開啟：

1. 開啟專案資料夾
2. 用 Live Server 啟動 `index.html`
3. 先新增產品與包材
4. 建立 BOM 對照
5. 測試出貨、進貨、盤點、排程與備份

目前不需要後端伺服器，也不需要資料庫。

## 資料儲存

資料存在瀏覽器 `localStorage`。

localStorage key：

```text
ims_v5
```

主要資料：

- `items`：產品與包材
- `bom`：產品與包材對照
- `logs`：操作紀錄
- `locations`：庫位資料
- `schedules`：出貨排程
- `nextId` / `nextSchId`：下一筆 ID

重要提醒：

- 清除瀏覽器資料會清掉系統資料
- 換瀏覽器或換電腦看不到原本資料
- 正式使用時請定期匯出 JSON 備份

## 基本測試清單

每次修改後建議確認：

- 新增 / 編輯 / 刪除品項正常
- 出貨會扣產品與 BOM 包材
- 包材不足時不能出貨
- 進貨會增加庫存
- 批次盤點會更新庫存
- 出貨排程只能扣庫一次
- 已出貨排程編輯後仍維持已出貨
- 庫位合計差異顯示正常
- 月報表與趨勢圖能顯示
- Excel 匯出正常
- JSON 備份與還原正常

## 開發原則

- 優先小範圍修改
- 不為了整理而大改 UI
- 新功能先確認資料格式
- 使用者輸入塞進 `innerHTML` 前要 escape
- 日期比較優先使用本地日期 helper
- 修改檔案前先確認應該動哪個模組
